day12总结

day11总结

1.final 关键字

  • final关键字可以修饰类，方法，变量
  • final修饰类不可以继承
  • 方法不可以被重写
  • 变量不能被重写
  • 扩展
    • final和abstract不可以同时修饰一个类和方法
    • static和abstract不可以同时修饰一个方法

2.接口

  • 接口的成员
    • 自定义常量
    • 抽象方法
    • 默认（8.0），
      • 子类继承实现父类和接口，父类和接口中的同名方法，调用子类对象执行的是父类的实例方法
      • 子类多继承接口时要重写接口中的重名方法
    • 静态（8.0）
      • 调用格式：接口名,静态方法名（）；
    • 私有（9.0）方法
  • 接口的注意事项
    • 接口不可以实例化对象，接口中没有构造器
    • 类与接口之间的关系
      • 类与类之间单继承，可以多层继承
      • 类与接口之间：可以多实现
      • 接口与接口：可以多继承

3.多态

  • 子类 name = new 子类 （） （单态）

  • 多态的前提条件

    • 要有继承/实现
    • 要有方法的重写(不重写没有意义,编译不会报错)
    • 父类的引用指向子类对象，父接口的引用指向实现类对象

  • 多态的格式

    • 继承关系：父 name=new 子类（）；
    • 实现：同上

  • 多态下各成员的特点

    • 实例变量

      • 多态和属性没有关系，被父类修饰就找父类的变量被子类修饰就找子类

    • 构造器

    • 与多态没有关系，先初始化父类，再找子类

    • 实例方法

    • 多态只和实例方法有关系，通过多态形式的对象进行方法的调用，先看父类/接口是否有这个方法，如果有，执行子类（实现类）重写之后的方法，如果没有编译报错。

  • 多态的好处

    • 讲父类型作为方法参数，只需要讲对应的子类型放入。
