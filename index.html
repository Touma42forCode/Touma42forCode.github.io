二、事务

1、事务的概念

事物式一组sql组成的单元，他们要么同时成功，要么同时失败。这组sql一般是完成一个完整独立的业务逻辑。

2、事物的ACID特性

  • 原子性 || 一致性|| 隔离性|| 永久性

3、mysql如何控制事务

​ 1.改自动提交为手动提交然后commit/rollback

​ 2自动提交下start transaction，执行sql ，然后commit/rollback

4、事务隔离性

（1）事务并发性问题

​ 藏独，不可重复读，幻读

（2）事务隔离级别

​ mysql默认隔离级别：repeatable-read

​ read-uncommitted:三种问题均会出现

​ read-committed:可以避免脏读

​ repeatable-read:理论上会出现幻读，但是mysql中使用多版本控制技术可以避免幻读

​ serializable：可以避免三种问题。效率低，安全性高。

1.泛型

  • 形式：< > 一对尖括号

  • 泛型是什么：泛化的类型，或者说是指类型参数

  • 在实现方法的功能时，有些数据时未知的，需要泛型

  • 泛型也有形式参数和实际参数，但是<>中填写的一定是数据类型，而且一定是引用数据类型。

  • <泛型形参>一般用大写字母，例如：

    ​ :type类型，泛指类型

    ​ element,元素类型

    ​ return type,返回值类型

    ​ argument，形参的类型

  • <泛型实参>一般写具体的类型

2. 为什么要使用泛型

避免类型转换，和类型转换错误

3.泛型类与泛型接口

  • 语法形式：

修饰符 class 类名 extends 父类 implements 父接口们{

关于高考想说的一些话 

高中时候老师总是会安慰大家：现在努力学，上了大学就可以随便玩了。从一个高中老师的角度来讲，这些话能给学生带来学习的动力。但是从长远的角度来讲，我认为这些话对于学生的成长是有害的。很多学生正是听信了高中老师的这番话（比如我），在大学放浪形骸，虚度终日，等到毕业的时候才猛然发现自己并没有掌握到足以在社会上生存的本领。
现在的我明白的一个道理是：人生的任何阶段都不应该放弃努力。不存在努力一段时间，努力到一定的程度，之后的日子就可以躺在功劳簿上吃老本。这样的现象确实是存在的，但是并不是一种健康的人生观。高中的时候全校演讲总有尖子生说：让优秀成为一种习惯，当时觉得好学生就会说漂亮话，一点也不实际。
我并不是一个力争上游的人，我只是想搞钱，赚到够我花的钱就是我的人生目标。优秀对于我来说太虚无。我可以在短时间内努力拼命达到一个目标然后垮掉，等到情况危机的时候再次拼命的努力。我的中考，高考都是这么过来的。这是我的基本生存技能。但是在大学这个条件反射失效了，因为大学时光实在太长了而且无拘无束了，没有任何眼前的忧愁迫使人往前走。堕落是多数人的常态。认为自己从监狱里逃出来的人肯定不会再向往监狱的生活。从小被管制到大的大一新生自然也不会认为自律的生活是一种多么可贵的生活习惯。
等到我找工作的时候我才发现四年来脑子空空，被游戏填满生活的我走入社会完全就是废人一个。

Day 13 笔记

1.设计模式

专门解决一类问题的思想。

单例设计模式：为了只创建唯一的对象，需要

​ 1.将类中的构造器私有化，

​ 2.在类中成员位置创建唯一对象，并用private static final修饰

​ 3.考虑到封装思想，需要将静态变量私有化，对外提供公共的访问方式 (饿汉式)

​ 1.类追踪的构造器私有化

​ 2.在类中成员位置定义（声明）一个这个类型的变量用private static修饰

​ 3.考虑到封装思想，将静态变量私有化，对外提供公共的访问方式

2.枚举类

格式：修饰符  enum  类名{

​ 对象名1，对象名2 }

​ 枚举类的特点：

​ 1.如果枚举类没有任何的构造器，JVM会提供一个无参的；不能被实例化；

​ 2.如果枚举类中的对象是通过无参构造器进行对象的创建，对象后面的小括号可以不写；

​ 完整格式：

​ 修饰符 enum 类名{

​ 对象名1（实参），对象名2（实参）….

​ }

​ 3.枚举类中对象名必须放到第一行

​ 4.枚举类中的对象其实用public static final修饰

​ 注意事项：

​ 1.在java中枚举类默认继承Enum类，不能再继承其他类，但是可以实现接口。可以使用Enum中国的所有方法

​ 2.可以实现接口

​ 3.普通类不能继承枚举类，理论上可以，但是普通类实现不了枚举类内特殊的语法，所以不能继承

枚举类常用方法：

​ toString（）；name（）；

​ ordinal（），返回常量的次序号；

​ value（）返回该枚举类的所有的常量对象；放回类型是当前枚举的数组类型

装箱与拆箱（已实现自动化）

字符串==>基本类型

    int name = new Integer（“”）；自动装箱

  • 包装类型的静态方法

    public static xxx parseInt（String s）
