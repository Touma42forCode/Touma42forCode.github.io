day11总结

day11总结

1.final 关键字

  • final关键字可以修饰类，方法，变量
  • final修饰类不可以继承
  • 方法不可以被重写
  • 变量不能被重写
  • 扩展
    • final和abstract不可以同时修饰一个类和方法
    • static和abstract不可以同时修饰一个方法

2.接口

  • 接口的成员
    • 自定义常量
    • 抽象方法
    • 默认（8.0），
      • 子类继承实现父类和接口，父类和接口中的同名方法，调用子类对象执行的是父类的实例方法
      • 子类多继承接口时要重写接口中的重名方法
    • 静态（8.0）
      • 调用格式：接口名,静态方法名（）；
    • 私有（9.0）方法
  • 接口的注意事项
    • 接口不可以实例化对象，接口中没有构造器
    • 类与接口之间的关系
      • 类与类之间单继承，可以多层继承
      • 类与接口之间：可以多实现
      • 接口与接口：可以多继承

3.多态

  • 子类 name = new 子类 （） （单态）

  • 多态的前提条件

    • 要有继承/实现
    • 要有方法的重写(不重写没有意义,编译不会报错)
    • 父类的引用指向子类对象，父接口的引用指向实现类对象

  • 多态的格式

    • 继承关系：父 name=new 子类（）；
    • 实现：同上

  • 多态下各成员的特点

    • 实例变量

      • 多态和属性没有关系，被父类修饰就找父类的变量被子类修饰就找子类

    • 构造器

    • 与多态没有关系，先初始化父类，再找子类

    • 实例方法

    • 多态只和实例方法有关系，通过多态形式的对象进行方法的调用，先看父类/接口是否有这个方法，如果有，执行子类（实现类）重写之后的方法，如果没有编译报错。

  • 多态的好处

    • 讲父类型作为方法参数，只需要讲对应的子类型放入。

day07 总结

day07 总结

1.1类的设计

1.2面向对象技巧

1.3类的设计

1.4实例变量和局部变量的区别：

  • 在代码中的位置不同

  • 在内存中的位置不同
  • 默认值不同
  • 在代码中作用域不同
  • 在内存中的生命周期不同

1.5 this关键字的第一种用法

  • 在类中的实例方法中用来区分重名实例变量和局部变量
  • 作用：哪个对象调用了this关键字所属的实例方法，这个this关键字就代表哪个对象

1.6 封装

  • 封装思想在代码中的体现就是权限修饰符
  • 权限从小到大：private<缺省<protected<public
  • 封装思想优化java类
    • 将类中的实例变量进行私有化（private关键字）
    • 针对私有化的实例变量提供访问方式：get/set方法

1.7 构造器

  • 作用：初始化对象，如果带参数的构造器可以给相应的实例变量进行赋值
  • 构造器的特点：
    • 构造器的名字与类名相同
    • 没有返回值类型，void也不能有
    • 构造器可以重载
    • 如果一个类没有构造器，jvm自动提供一个默认无参的构造器
    • 如果有类中有构造器，jvm不会再进行提供
